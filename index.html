用原版系统看pdf很纠结，在网上找了个可行的转换方法。在这里分享下。
原帖地址
http://www.mobileread.com/forums/showthread.php?t=90618

方法1：
To be able to convert pdf files in mobi format, I have taken the following steps:
1. convert pdf file into word document ( I’ve used Nuance PDF converter)
2. save word document as rtf file
3. convert rtf to mobi format using calibre

方法2：
还有一种方法就是用briss裁边，然后就直接看吧。想用calibre转也成。
briss
http://sourceforge.net/projects/briss/files/release%200.0.11/briss-0.0.11-bin.zip/download

方法3：
用acrobat转为html。然后把html交给calibre做最后处理。
感觉这个方法是目前最靠谱的一种。毕竟adobe自己的软件还是支持最好的。此法基本克服了calibre直接转死慢和多数不兼容的问题。。。

用命令方式为NAS添加第二块硬盘

发表于 2012-10-13   |   分类于 NAS   |  

之前我的buffalo的NAS是单硬盘工作模式，因为LS-WVL可以支持两块硬盘，所以试着添加第二块。
为了保留之前的数据，所以没准备做Raid0模式-仅仅是速度性能，安全稳定性太差，而且一旦有点问题数据皆毁。Raid1模式个人也觉得没有必要，要数据安全还是要对地备份:-)。
因此就直接普通的方式挂载上去，这里并不相应buffalo自带的web工具进行分区格式化，一个是效率低，操作不当还容易毁掉数据(变raid0/1)。
当然首先机器要Root了，否则只能用自带的了，先用160G的做实验。：
1. 关机，在空位插上硬盘
2. 开机，系统一切正常
3. parted -l 查看系统是否正确找到了第二块硬盘，应该可以看到/dev/sdb
(-s 参数很重要，如果不加入，则会是一个交互式操作)
4. parted -s /dev/sdb mklabel gpt
5. parted -s /dev/sdb mkpart primary 1 160G
6. parted -s /dev/sdb print
7. mkfs.xfs -f /dev/sdb1
8. mkdir /mnt/disk2
9. mount /dev/sdb1 /mnt/disk2

下一步，准备自动mount-编辑/etc/fstab。参考http://hi.baidu.com/t8j8x8/item/06a5ba35a31afe667d034b38

windows 共享名问题处理

发表于 2012-10-05   |   分类于 Win   |  

在处理共享目录时时常会遇到切换用户的问题，如果打开了，windows系统会自动保持(如果之前选择自动记录)

删除共享目录的链接
net use \ip\目录 /del

建立共享目录的非空链接
net use \ip\目录 “密码” /user:”用户名”

查看有哪些用户
net user

其它一些用法：
net use \ip\ipc$ “ “ /user:” “ 建立IPC空链接
net use h: \ip\c$ “密码” /user:”用户名” 直接登陆后映射对方C：到本地为H:
net use h: \ip\c$ 登陆后映射对方C：到本地为H:
net use h: /del 删除映射对方到本地的为H:的映射
net user 用户名　密码　/add 建立用户
net user guest /active:yes 激活guest用户
net user 帐户名 查看帐户的属性

Buffalo LS系列系统XFS文件系统及读取备份

发表于 2012-10-03   |   分类于 NAS   |  

之前在nas1论坛发的帖子，搬过来放在自己的系统上:)

原文如下：

已经找到解决方案了(我的是单盘的)，献给有需要的NAS迷：
1. 找个PC (虚拟机也一样，我就是) 安装 Ubuntu 12.04, 只要支持 xfs文件系统就可以
2. 拔下硬盘，连接到PC上(我用的USB转接口)，直接接到主板的Sata口速度效果更好
3. sudo Parted -l 查下刚插上的硬盘 (假设 /dev/sdb)
4. sudo mount /dev/sdb6 (假设你要挂第6个分区)
5. 如果成功，到资源管理器里面找刚挂上的盘吧，就复制你的数据吧（最好使用Root 账号操作 拷贝）
6. 如果失败(提示unknown filesystem type ‘linux_raid_member’ ), 查下是否安装了mdadm Raid磁盘管理工具。sudo apt-get install mdadm
7. sudo mdadm —assemble —run /dev/md0 /dev/sdb6
8. 如果正常退出，ok，顺利完成。
9. 资源管理器里面找你的盘吧，备份你的数据吧。
—PS. 如果你的是双硬盘（Raid0，Raid1),甚至是Raid 5，原理一样，就是在PC上用mdadm工具把你多个硬盘挂在起来。因为本人只用一块，时间有限，没做实验。有成功者，update下哦。

*对于期望无损转换，觉得不好弄，如果可以实现，也是速度超超级慢的，还不如这样方便。

顺便问下，为什么 buffalo wvl 上我插的是一个硬盘，磁盘分区信息是raid 1 (因为如果你用df -h 查的话，硬盘文件符是 /dev/md1),而且在PC上用dmadm查也是显示raid level = 1。
在web控制页面里是显示一个磁盘，且是normal的。
纳闷呀，那我插入第二块硬盘，岂不乱套了？

有哪位高人解释下？ 插入个200G的硬盘会不会有问题？

论坛有人回复：“不会有任何问题，因为第一个硬盘中，已经被WVL分割了好几个磁区，你的资料只放在其中一个磁区。
第二个硬盘再插入，在不做RAID的情况下，就只会有一个磁区。”
保险期间有空还是先实验下。

搜索的一些参考资料：
1. Q: support XFS?
A: modprobe -l | grep -i xfs
2. mdadm for raid
http://serverfault.com/questions/383362/mount-unknown-filesystem-type-linux-raid-member
You should not mount it directly using mount. You need first to run mdadm to assemble the raid array. A command like this should do it:
$ mdadm —assemble —run /dev/md0 /dev/sdc1
If it refuses to run the array because it will be degraded, then you can use —force option. This is assuming you don’t have /dev/md0 device. Otherwise, you need to change this name.
When this command is executed successfully, you can mount the created device normally using:
$ mount /dev/md0 /mnt/test
